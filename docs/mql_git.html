Git

These are the functions that will be use to do git operations.

git_addgit_commitgit_pushgit_clonegit_pull

git_add <repopath> <directories>

Description

    This function will add untracked files from your git repo for an eventual push/commit later.

Parameters

    repopath:   The path to your local repository. - string - required
    directories:   The directories in your repo that will be added. Put '.' to add every directories within the repository. - string - required
admin
mentdb

git_commit <repopath> <message>

Description

    This function will save every local tracked files that you saved with git add, also this function needs a message that will describe the commit you made. (It is recommended to write concise and comprehensive commit message so you know why you did a commit at this moment).

Parameters

    repopath:   The path to your local repository. - string - required
    message:   The message that will be associated to the commit you have made. - string - required
admin
mentdb

git_push <repopath>

Description

    This function will push to the branch specified to the host repo. The standard branch on a repo is always the 'master' branch.

Parameters

    repopath:   Path to your local repository. - string - required
admin
mentdb

git_clone <repopath_local> <user> <ip> <port> <repopath_serv>

Description

    This function will make a clone of the repository that is present on the distant host to your computer. You can specify the path for where you want to have the clone to be located.

Parameters

    repopath_local:   The path to the repo where the clone will be. - string - required
    user:   The user that you are trying to connect to. - string - required
    ip:   The ip of this user. - string - required
    port:   The port of the server. - string - required
    repopath_serv:   The path to the distant repo. - string - required
admin
mentdb

git_pull <repopath>

Description

    This function will update your local repo to match the state of the distant repo. It is mainly use to maintain your repo update when someone else pushes something on the repo.

Parameters

    repopath:   The path to your local repo. - string - required
admin
